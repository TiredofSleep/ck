## §1. What this atlas is (and is not)

> **This atlas is not a unified proof document. It is a memory-preserving synthesis scaffold: exact results, structured reductions, bridge attempts, preserved failures, and candidate maps are held together here without overstating which links are proved.**

**Explicitly:**

| NOT | YES |
|---|---|
| Not one proof | A catalog of local charts, thresholds, defect sets |
| Not one unified theorem | Reduction ladders and bridge attempts |
| Not a claim that all threads are already bridged | Open maps preserved with named gaps |
| Not a prediction engine | A disciplined synthesis layer above independent theorem tracks |
| Not a speculation accumulator | A memory object that keeps discoveries from being forgotten |

Each item in the atlas carries an **epistemic flag**:

- **[fire]** — verifiable, reproducible, proved
- **[gold-with-gap]** — result with one or more named open pieces
- **[speculative]** — suggestive but not bridged; preserved, not promoted
- **[caution]** — past overclaim; preserved with context so errors don't propagate

A claim without a flag has not yet been flagged and should be treated with suspicion.

## §5. The recognitions framing (corrected stance)

**The atlas has undergone one methodological correction between earlier drafts and v3.5. Read this before reading §10 (Clay attacks).**

**Earlier framing:** The Clay problems are crossings — construction projects where a map is built from the algebraic side (T* = 5/7) to the analytic side (Re(s) = 1/2).

**Corrected framing (FINAL_REDUCTION.md in-document correction, 2026-04-03):** The Clay problems are **recognitions** — they ask whether two lenses (structure: Z/10Z, finite, committed; flow: analytic, infinite, open) are measuring the same threshold.

**Why the correction matters:**

- "Crossings" sounds like construction. Implies Clay = do work.
- "Recognitions" sounds like observation. Implies Clay = notice structure that's already there.

The crossings-framing was aspirational. The recognitions-framing is the one the work has actually matured into.

Atlas v3.5 §17 documents this correction fully. §10 body still uses "crossings" vocabulary in places — the correction is flagged throughout (§15 caution #8, §17 full reframe) but the textual update pending `DUAL_LENS_CLAY.md` surfacing.

**Reader instruction:** when reading §10, substitute "recognition of lens-coherence" wherever "crossing" appears, until the body is rewritten in v4.

## §6. Thread separation is architecture, not fear

The atlas maintains three separate threads: **PPM / Hodge / Q-series.** The separation rule is absolute (no composite claims, no cross-thread bridging, no unified narrative that threads all three).

**Why this is architecture, not defensiveness:**

> **Independent theorem tracks are preserved below; disciplined synthesis happens above them.**

- **Below:** each thread has its own rigorous theorem track with its own vocabulary, its own standards, its own verifiers, its own failure modes. PPM verdicts don't cite Hodge sprint results; Hodge sprints don't cite PPM verdicts; Q-series stays in Q-series vocabulary.
- **Above:** the atlas is the synthesis layer. It notes where threads exhibit parallel structures (e.g., "suggestive, not bridged" language). It does not merge threads.

**The alternative** (merging threads into one unified narrative) would:
- Create composite claims that aren't individually verifiable
- Lose the ability to roll back one thread when another has issues
- Bury failure modes that should stay visible
- Overstate bridges that haven't been built

**The discipline, restated positively:** the atlas lets each thread reach its own honest conclusion. The synthesis layer above is descriptive, not prescriptive — it describes what parallels exist, not what bridges should be forced.

**This is what makes the atlas externally credible.** External reviewers trust a document that separates threads more than one that merges them.

## §7. What really overlaps — disciplined ranking

Not all overlaps across the atlas are equal. This ranking keeps synthesis honest:

- **Z/10Z local chart ↔ PPM:** both operate on seams in finite grammars; seam = MAX/MIN in both.

- **CK ↔ theorem program at architecture level:** NOT a theorem bridge — an architecture bridge (both use layered identity / recovery / provenance).

### Suggestive overlap (interesting, not bridged)

- **PPM / Hodge / Q-series via "multiplicative loading":** the P3AP carrier-family property in PPM, the Galois conjugation σ: i ↦ −i in Hodge W_*, the g=3 primitive root in Q-series — all share a "multiplicative structure imposes directional asymmetry" flavor. Not bridged. Suggestive only.
- **Li Foundation threshold ↔ RH corridor ↔ bridge width:** λ_n ≥ T* in Li, RH ⟺ m_KV > 0 uniform in Halving Lemma, bridge width 3/14 in FINAL_REDUCTION — all three are parallel restatements of "RH as finite-dimensional threshold-hold." Parallel, not bridged.
- **Shape-admissibility ↔ observed low-degree structures:** SAH compatible with the low-degree corridor structures observed in PPM and the low-rank algebraic structures in Hodge W_*. "Compatible with" is the exact permitted phrasing (§8.5 of master atlas). Nothing stronger.

### Not yet bridged (open research questions)

- **Formal operator map from Z/10Z to Hodge obstruction.** Would require translating W_* block structure into Z/10Z-derived terms. Not done.
- **Formal bridge from PPM seam logic to Clay analytic objects.** Would require embedding Z/10 seams into ζ or L-function infrastructure. Not done.

**Reader instruction:** when you see a claim in the atlas that parallels two domains, check which bucket it falls into. Only "strong overlap" justifies structural claims. "Suggestive" and "not yet bridged" justify speculation only.
