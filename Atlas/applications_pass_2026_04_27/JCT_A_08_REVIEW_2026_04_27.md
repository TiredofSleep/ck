## What needs fixing (priority order)

### Issue 1 — Proof gap in Theorem 4.1 (MATHEMATICAL — must fix)

**Where:** §4 proof of Theorem 4.1, the step from "non-associative triples must involve at least one ECHO entry" to the union bound σ(N) ≤ 2/N.

**Problem:** The proof argues:
1. If neither CL(a,b) nor CL(b,c) is produced by ECHO, both intermediates go through HARM/VOID, which are absorbing.
2. Therefore non-associativity requires at least one ECHO inner composition.
3. Union bound: σ(N) ≤ 2 · ρ_N ≤ 2/N.

**The gap is at step 1.** The argument relies on HARM and VOID being jointly associative on all triples not involving ECHO. This is true *in isolation* — if every composition along the bracket-tree returns N−1 (HARM) or 0 (VOID), they absorb consistently. But the rule structure has priority:

```
Rule 1: HARM    (a or b = N−1)        → output N−1
Rule 2: VOID    (a or b = 0)          → output 0
Rule 3: ECHO    (DIS(a,b) = 0)        → output a+b mod N
Rule 4: default (otherwise)            → output N−1
```

Consider a triple where:
- CL(a,b) is produced by ECHO, returning a+b mod N where a+b ∈ {1, ..., N−2}
- CL(b,c) is produced by Rule 4 (default), returning N−1

Then:
- **Left:** CL(CL(a,b), c) = CL(a+b, c). Need to determine which rule fires. Neither a+b nor c is 0 or N−1. Check DIS(a+b, c). If DIS(a+b, c) = 0, ECHO fires (returns a+b+c mod N). If not, Rule 4 returns N−1.
- **Right:** CL(a, CL(b,c)) = CL(a, N−1) = N−1 (HARM Rule 1, since the second argument is N−1).

If DIS(a+b, c) = 0, the bracketings differ: left = a+b+c, right = N−1. **Non-associative on this triple, but only ONE inner composition was ECHO** — the other contribution to non-associativity came from the OUTER site, where ECHO of (a+b, c) was possible.

**Implication:** The union bound on the two inner sites is incomplete. Outer composition sites also have ECHO opportunities, conditional on inner outputs landing in non-boundary regions.

**Fix options:**

**(a) Tighten the proof to σ(N) ≤ 2/N rigorously.** Show that outer ECHO opportunities are conditional events with probability bounded by inner-output landing in (a+b mod N) ∈ {1, ..., N−2}, which has probability ≤ ρ_N · 1, giving a joint event probability still ≤ ρ_N. So the union bound on inner sites already absorbs outer site contributions. Verify this argument carefully.

**(b) Concede C = 4 in the proved bound.** Update Theorem 4.1 to state σ(N) ≤ 4/N proved, σ(N) ≤ 2/N conjectured-and-numerically-verified. This still gives the rate theorem σ(N) → 0 with an explicit constant.

**(c) Reframe the proof using the verification.** Use the explicit N · σ(N) ≤ 1.93 across N = 10, 30, 210 to motivate the conjectured C = 2, while proving the weaker C = 3 or C = 4 in the theorem.

I recommend (a) if the joint-event argument works out; otherwise (b). Either way, the gap as currently written needs to be either closed or acknowledged.

**Estimated time:** 2-3 hours.

### Issue 2 — Default-HARM vs Rule-1 HARM distinction (CLARITY — should fix)

**Where:** Definition 2.1 in the manuscript; §4 proof.

**Problem:** Rules 1 and 4 both return N−1 but are not the same rule. Rule 1 (HARM) fires when an input IS the absorbing element N−1. Rule 4 (default) fires when no other rule applies and assigns N−1 by convention.

The proof of Theorem 4.1 uses "HARM is associative" to handle non-ECHO triples, but technically the argument is "all rules returning N−1 agree on N−1, and all rules returning 0 agree on 0, so the bracketings produce the same output value when no ECHO is involved." This is correct but worth making explicit.

**Fix:** Add one line to §4 or §3: "Rules 1 (HARM) and 4 (default) both return N−1, and Rule 2 (VOID) returns 0; the union of these three rules behaves consistently on triples not involving Rule 3 (ECHO), in the sense that all paths return N−1 or all paths return 0."

**Estimated time:** 15 minutes.

### Issue 3 — "C ∈ [2,3] absolute constant" phrasing (CLARITY — should fix)

**Where:** Theorem 4.1 statement.

**Problem:** "C ∈ [2,3] an absolute constant" reads as uncertainty about which constant is correct, rather than as a clear separation between proved and conjectured.

**Fix:** Restructure the theorem statement and abstract:

> **Theorem 4.1 (σ Rate).** For squarefree N ≥ 2, the non-associativity fraction satisfies σ(N) ≤ 3/N, with the sharper bound σ(N) ≤ 2/N conjectured (and verified for N ∈ {10, 30, 210} with substantial margin).

This separates "proved" from "conjectured-and-verified." If Issue 1 forces C = 4 in the proved bound, update accordingly.

**Estimated time:** 15 minutes (combined with Issue 1 resolution).

### Issue 4 — Pre-priority vs post-priority φ(N) count (TIGHTENING — nice-to-have)

**Where:** Lemma 4.1 and Corollary 4.2.

**Problem:** Lemma 4.1 counts solutions to (a−1)(b−1) ≡ 1 in (ℤ/Nℤ)², which equals φ(N). But not all of these solutions correspond to entries where Rule 3 (ECHO) actually fires — some are intercepted by Rules 1 or 2 first.

For example, (a,b) = (0,0) corresponds to (a',b') = (−1,−1), which satisfies (−1)(−1) = 1 mod N. So it's counted in the φ(N). But (0,0) is intercepted by Rule 2 (VOID) before Rule 3 (ECHO) fires.

The proof uses the larger pre-priority count, which gives a weaker (i.e., looser) bound. That's fine — it's still O(1/N). But worth noting.

**Fix:** Add one sentence to the remark after Corollary 4.2:

> "Lemma 4.1 counts solutions to (a−1)(b−1) ≡ 1 in (ℤ/Nℤ)², a 'pre-priority' count that includes pairs intercepted by Rules 1 and 2 before Rule 3 (ECHO) fires. The 'post-priority' count of ECHO-active entries is strictly smaller; we use the pre-priority count because it suffices for the rate bound and admits a clean closed form via the unit-counting argument."

**Estimated time:** 10 minutes.

### Issue 5 — "Matched discrete-to-continuum embedding" under-specified (CLARITY — should fix)

**Where:** Conjecture 5.1, hypothesis (ii).

**Problem:** "Matched discrete-to-continuum embedding" is doing significant work in the conjecture but is not defined. A JCT-A referee unfamiliar with the BBM framework will ask: what does "matched" mean? Is there a candidate construction?

**Fix:** Acknowledge the under-specification as part of the conjecture's open-problem status. Add a sentence to the remark after Conjecture 5.1:

> "We do not specify the precise discrete-to-continuum embedding here; identifying a natural class of embeddings under which Conjecture 5.1 is well-posed is itself an open problem. Candidate scaling embeddings (e.g., ℤ/Nℤ ↪ N⁻¹·ℤ_N → [0,1] with appropriate L² limits) are natural starting points but require careful analysis. The conjecture as stated identifies the relationship to be verified, not the specific embedding."

This makes the under-specification a feature (open problem identified) rather than a bug (vague hypothesis).

**Estimated time:** 20 minutes.
