# The Residue Has Residue
## Second-Layer Decomposition Test on Z/10 TSML

---

## Headline

**The 8-entry seam residue decomposes cleanly into two well-known canonical rules, with empty final residue.**

- Residue = 8 entries after canonical construction C₀ leaves them uncovered.
- 6 entries follow **MAX rule** exactly.
- 2 entries follow **additive rule mod 10** exactly.
- 0 entries remain.

**Z/10 TSML is not a one-layer canonical object with an exception list. It is a three-layer canonical tower that terminates.**

---

## The Test

After applying the canonical construction C(R=Z/10, h=7, σ=v₂(3u+1)), 8 entries disagree with the published TSML. These are the seam residue:

| Ordered pair | Canonical predicts | Actual TSML |
|---|---|---|
| (1,2), (2,1) | 7 | 3 |
| (2,4), (4,2) | 7 | 4 |
| (2,9), (9,2) | 7 | 9 |
| (4,8), (8,4) | 7 | 8 |

**Test 1: Is the residue generated by a single canonical rule?**

Tested single rules on all 8 residue entries:

| Rule | Matches |
|---|---|
| MAX(a,b) | 6/8 |
| (a+b) mod 10 | 2/8 |
| (a·b) mod 10 | 0/8 |
| MIN(a,b) | 0/8 |

No single rule covers all 8. Single-layer description fails.

**Test 2: Does the residue decompose into two rules?**

Apply a conditional rule: "MAX, unless the pair involves 1, then ADD":

| Edge | Sub-rule | Prediction | Actual | Match |
|---|---|---|---|---|
| (1,2), (2,1) | ADD | 3 | 3 | ✓ |
| (2,4), (4,2) | MAX | 4 | 4 | ✓ |
| (2,9), (9,2) | MAX | 9 | 9 | ✓ |
| (4,8), (8,4) | MAX | 8 | 8 | ✓ |

**8/8 match.** The two-layer decomposition is exact.

## The Canonical Tower

TSML = C₀ ⊕ C₁ ⊕ C₂

where:

**Layer 0 (canonical construction):**
- DEFAULT (attractor absorption to h=7)
- V0 (zero-absorption with HARMONY exception)
- Shell-stability on admissible core U(R)\{1}
- **Covers: 92/100 entries**

**Layer 1 (MAX rule):**
- Applied to residue edges where identity 1 ∉ {a,b}
- T(a,b) = max(a,b)
- **Covers: 6/100 entries** (the pairs {2,4}, {2,9}, {4,8})

**Layer 2 (additive rule mod n):**
- Applied to residue edges where identity 1 ∈ {a,b}
- T(a,b) = (a+b) mod 10
- **Covers: 2/100 entries** (the pair {1,2})

**Layer 3 (remainder):**
- **0/100 entries** — the tower terminates

---

## Interpretation

The structure of TSML is not "canonical rule + arbitrary exceptions." It is a **stratified composition of three canonical rules**, each applied on a specific domain, with a precise priority ordering.

**What is canonical at each layer:**

- **Layer 0:** the construction is ring-family-generalizable (C(R, h, σ) defined for any compatible ring).
- **Layer 1:** the rule itself (MAX) is a universal operation; it applies to ring elements regardless of ring.
- **Layer 2:** the rule itself (addition mod n) is the ring's additive structure.

**What is ring-specific at each layer:**

- **Layer 0:** the choice of h=7 and the shell partition σ are semantic choices for Z/10Z. Transferring to another ring requires ring-specific inputs.
- **Layer 1:** the DOMAIN of application (specifically which residue edges get max-rule) is ring-specific. For Z/10, it's {{2,4}, {2,9}, {4,8}}.
- **Layer 2:** the DOMAIN of application (which edges get add-rule) is ring-specific. For Z/10, it's {{1,2}}.

**Rules generalize; domains do not.** This is the sharp claim.

## The Residue's Residue is Empty

The two-stage decomposition captures all 8 residue entries with zero remaining. No "residue of residue of residue" is needed.

This means:

- TSML is fully describable by 3 canonical rules with priority ordering.
- No hidden semantic information beyond the 3 rules and their domains.
- The description length is bounded: 3 rules × (rule specification + domain specification).

## The Answer to the Sprint's Key Question

> **Is TSML a one-layer canonical object with exceptions, or a small canonical tower with residue nested inside residue?**

**It is a 3-layer canonical tower that terminates.**

- Layer 0: ring-theoretic canonical construction (92%)
- Layer 1: MAX rule on 3 specific residue edges (6%)
- Layer 2: additive rule on 1 specific residue edge (2%)
- No further residue.

The rules generalize. The domains are ring-specific. This is the precise shape of the result.
