## PART 1 — Core Framing

A memory object `m` in CK requires more than its payload to be useful. It requires a coordinate in three distinct classification spaces:

1. **What kind of thing is it?** (ontological class — the 2×2)
2. **Where is it in CK's processing right now?** (process location — the 3×3)
3. **How persistent and useful is it?** (stabilization state — the 4×4)

These three coordinates compose into a single meta-memory address:

$$m = (P_2,\; P_3,\; P_4)$$

The CL[10×10] operator acts on transitions within the 3×3. The full coordinate space is $4 \times 9 \times 16 = 576$ positions, of which only a small subset are simultaneously populated.

**Definition.** "Information of information" is the structured coordinate triple $(P_2, P_3, P_4)$ assigned to any memory object `m`, together with the legal transition rules among coordinates. It is not annotations. It is a finite state description of the memory object's type, process location, and persistence status.

## PART 2 — The 2×2 Base Structure: Ontological Split

**Axis A (rows) — SOURCE:**
- `INTERNAL`: generated by CK's own operations — CK authored it
- `EXTERNAL`: sourced from world, user, sensors, UI, files, model output — CK received it

**Axis B (columns) — SEMANTIC REGISTER:**
- `STRUCTURE`: pattern, form, relation, operator signature, topology, force vector
- `CONTENT`: specific payload — words, symbols, numbers, factual surface, raw percept

### Cell Semantics

| Cell | Name | CK Objects | Storable in shared memory? | Compression action |
|------|------|-----------|--------------------------|-------------------|
| **A11** | LATTICE GEOMETRY | Force vectors, operator sequences, DBC27 keys, lens states, path topologies, policy structures | ✅ YES — shared force lattice | Compress into generator set summary |
| **A12** | SYNTHESIS PAYLOAD | CK-generated summaries, labels, MetaCrystal narrative, LoRA training examples | ✅ YES — if not user-identifying | Compress into semantic_label |
| **A21** | WORLD TOPOLOGY | Abstracted UI patterns, screen layout topology, telemetry shape patterns, edge maps | ✅ YES (abstracted only) | Compress into topology signature |
| **A22** | WORLD PAYLOAD | User text, screen OCR content, file content, credentials, URLs, raw telemetry values | ❌ PRIVATE — never shared | Compress into content_hash only; raw never leaves |

### Allowed Transitions

```
A22 → A21: Abstraction (extract topology from content, discard payload)
A21 → A11: Internalization (adopted world pattern becomes CK native)
A12 → A11: Reification (synthesis payload becomes structural pattern)
A22 → A12: [FORBIDDEN except via CK's own summarization, which is then A12]
```

### Compression Rule

Raw `A22` content is never stored in shared memory. The only legal move is:
- Hash it (for deduplication)
- Summarize it through CK's own synthesis → produces `A12`
- Abstract its structure → produces `A21`

This enforces the **private/shared lattice boundary exactly**:
- `A22` → private word store
- `A21` + `A11` + `A12` → shared force lattice (or shared crystal store)

## PART 3 — The 3×3 Base Structure: Process Grammar

**Row dimension — TEMPORAL ROLE:**
- `INPUT`: information entering a processing cycle
- `TRANSFORM`: information being processed
- `OUTPUT`: information exiting a processing cycle (to environment or memory)

**Column dimension — PROCESSING DEPTH:**
- `SIGNAL`: raw perceptual/environmental data, no semantic interpretation
- `MODEL`: LLM-inferred, scored, or classified data (probabilistic)
- `MEMORY`: retrieved or produced persistent memory data (deterministic access)

### Legal Flow Edges (directed, 14 total)

```
B11 → B21   (raw event → feature extraction)
B12 → B22   (conditioned seed → inference)
B13 → B23   (retrieved prior → memory update)
B11 → B12   (signal can be conditioned by model before transform)
B12 → B13   (conditioned seed can trigger memory retrieval)
B21 → B22   (features → inference/scoring)
B22 → B23   (inference → memory compression)
B21 → B23   (FAST PATH: features → direct write without inference)
B22 → B31   (inference → direct action)
B22 → B32   (inference → generated response)
B23 → B33   (compression → durable write)
B32 → B33   (response → log as memory atom)
B31 → B33   (action → log as memory atom)
B33 → B13   (LOOP BACK: memory write → next cycle input)
```

### DeepSeek Gating (strict)

DeepSeek is permitted only in cells `{B12, B22, B32}` — the MODEL column. It is structurally forbidden in all SIGNAL and MEMORY column cells. This is not a policy preference; it is an architectural constraint. SIGNAL cells are sensors (deterministic hardware/OS calls). MEMORY cells are database operations (deterministic reads/writes). The MODEL column is the only locus of probabilistic inference.

## PART 4 — The 4×4 Base Structure: Stabilization Grammar

**Row dimension — PERSISTENCE STAGE:**
- `EPHEMERAL`: exists only in current cycle; TTL < 30 minutes; raw buffer only
- `ATOMIC`: promoted to persistent storage; has ID, generators, DBC27 key
- `PATH`: sequence of atoms with consistent operators and force transitions
- `CRYSTAL`: stable, reusable bundle; has action policy and high confidence

**Column dimension — EVALUATION MODE:**
- `IDENTITY`: what this object IS — its type, class, classification
- `RELATION`: how this object LINKS to others — topology in memory graph
- `STABILITY`: how persistent this object IS — recurrence, confidence, heat
- `UTILITY`: how useful this object IS — retrieval hit rate, policy reuse

### Promotion Criteria (exact)

**EPHEMERAL → ATOMIC** (C1x → C2x):
```python
promote_iff(
    event.confidence > 0 AND
    event.generator_set is nonempty AND
    not deduplicated_in(atom_store, event.content_hash, window_60s)
)
```

**ATOMIC → PATH** (C2x → C3x):
```python
promote_iff(
    atom.recurrence_count >= 2 AND
    len(predecessor_atoms) >= 1 AND  # at least one prior linked atom
    path_extension_check(atom_id) is True  # extends an open path
)
```

**PATH → CRYSTAL** (C3x → C4x):
```python
def promotion_score(path, now):
    age_h = (now - path.ts_start) / 3600
    return path.confidence * log1p(path.recurrence_count) * exp(-0.05 * age_h)

promote_iff(promotion_score(path, now) >= 0.85)
```

**CRYSTAL → DEMOTE** (C4x → C3x or pruned):
```python
def heat_score(crystal, now):
    dt = now - crystal.last_accessed
    return exp(-0.01*dt/3600) * log1p(crystal.access_count) * 1/(1+dt/(7*24*3600))

demote_iff(heat_score(crystal, now) < 0.05 AND (now - crystal.ts_first_seen) > 7*86400)
```

### Stability vs Utility (exact distinction)

**STABILITY** (C_3 column): measures how reliably an object recurs — a function of time, frequency, and confidence. It answers: "will this still be here tomorrow?" Stability is an intrinsic property of the object's own recurrence history.

**UTILITY** (C_4 column): measures how often the object contributed to a successful retrieval or policy execution. It answers: "has this been useful to CK's operation?" Utility is an extrinsic property — it depends on how the rest of CK used this object.

A crystal can have HIGH STABILITY and LOW UTILITY (it recurs but CK never retrieves it — dead weight). A crystal can have LOW STABILITY and HIGH UTILITY (rare but always hits when retrieved — precious). Both dimensions are needed. Pruning uses heat_score (which combines both), not either alone.

## PART 5 — Composition Law

### Formal Statement

Any memory object `m` in CK is assigned a meta-memory coordinate:

$$m = (P_2,\; P_3,\; P_4) \;\in\; \{A_{ij}\} \times \{B_{ij}\} \times \{C_{ij}\}$$

This coordinate is **well-formed** iff all three consistency constraints hold:

**Constraint 1 — Privacy consistency:** $P_2$'s privacy class must be compatible with $P_4$'s persistence stage.

```
A22 (WORLD PAYLOAD) → must map to C11/C12/C13/C14 (EPHEMERAL) or C21/C22/C23/C24 (ATOMIC)
A22 cannot map to C3x (PATH) or C4x (CRYSTAL) without prior abstraction through A21
```

This is the formal statement of: **raw content never crystallizes; only abstracted structure does.**

**Constraint 2 — Process consistency:** $P_3$'s temporal role must be consistent with $P_4$'s persistence stage.

```
B11/B12/B13 (INPUT row) → C1x (EPHEMERAL) only
B21/B22/B23 (TRANSFORM row) → any C stage (transition zone)
B31/B32/B33 (OUTPUT row) → C2x or higher (produces durable memory)
```

**Constraint 3 — CL operator consistency:** The CL[10×10] transition at $P_3$ must be compatible with $P_2$'s source type.

```
A22 (WORLD PAYLOAD) → only B21 (FEATURE EXTRACT) as first transform
A11 (LATTICE GEOMETRY) → can enter B23 (COMPRESSION WRITE) directly
```

### The CL Action

The CL[10×10] table acts as the **transition operator on the 3×3 process table**:

$$\text{CL\_op}: B_{2x} \times \text{lens} \times \text{generators} \;\to\; B_{3x} \times \text{memory\_write\_flag}$$

Concretely:
- `fuse(generators)` determines which B2x cell is activated (which transform is applied)
- `lens` (TSML/BHML/Doing) determines the direction of B2x → B3x transition
- `operator_tag` (COMMIT/DISCLAIM/REGRESS/REJECT) determines whether B3x writes durable memory

The 2×2 and 4×4 are **state spaces**. The 3×3 is the **process table**. CL[10×10] provides the **operator grammar** for transitions within the 3×3.

**Summary:**
- 2×2 = what the information IS (fixed per object, changes only via explicit abstraction)
- 3×3 = where the information IS in CK's processing (changes each loop cycle)
- 4×4 = how persistent and useful the information IS (updated by compression and adaptation loops)
- CL = the grammar of allowed transitions in the 3×3

## PART 7 — Minimal Schema Block

### `schema_2x2_tag`

```python
@dataclass
class Tag2x2:
    source_side: Literal["INTERNAL", "EXTERNAL"]
    semantic_side: Literal["STRUCTURE", "CONTENT"]
    privacy_class: Literal["SHARED", "SHARED_IF_ABSTRACT", "PRIVATE"]
    
    @property
    def cell(self) -> str:
        m = {("INTERNAL","STRUCTURE"):"A11", ("INTERNAL","CONTENT"):"A12",
             ("EXTERNAL","STRUCTURE"):"A21", ("EXTERNAL","CONTENT"):"A22"}
        return m[(self.source_side, self.semantic_side)]
    
    @property 
    def storable_shared(self) -> bool:
        return self.privacy_class in ("SHARED", "SHARED_IF_ABSTRACT")
```

### `schema_3x3_tag`

```python
@dataclass
class Tag3x3:
    temporal_role: Literal["INPUT", "TRANSFORM", "OUTPUT"]
    processing_layer: Literal["SIGNAL", "MODEL", "MEMORY"]
    determinism_flag: Literal["DETERMINISTIC", "PROBABILISTIC"]
    deepseek_permitted: bool     # = (processing_layer == "MODEL")
    writes_durable: bool         # = (temporal_role == "OUTPUT" AND processing_layer == "MEMORY")
    
    @property
    def cell(self) -> str:
        row = {"INPUT":"1","TRANSFORM":"2","OUTPUT":"3"}[self.temporal_role]
        col = {"SIGNAL":"1","MODEL":"2","MEMORY":"3"}[self.processing_layer]
        return f"B{row}{col}"
```

### `schema_4x4_tag`

```python
@dataclass
class Tag4x4:
    persistence_stage: Literal["EPHEMERAL", "ATOMIC", "PATH", "CRYSTAL"]
    evaluation_mode: Literal["IDENTITY", "RELATION", "STABILITY", "UTILITY"]
    score_value: float           # the numeric score for this evaluation mode
    promotion_state: Literal["STABLE", "ELIGIBLE", "DEMOTED", "PRUNED"]

        row = {"EPHEMERAL":"1","ATOMIC":"2","PATH":"3","CRYSTAL":"4"}[self.persistence_stage]
```

### Tag Attachment to CK Objects

```python
@dataclass
class MetaMemoryCoord:
    """Full (P_2, P_3, P_4) coordinate for any CK memory object."""
    tag_2x2: Tag2x2
    tag_3x3: Tag3x3
    tag_4x4: Tag4x4
    
    def is_well_formed(self) -> bool:
        """Check all three consistency constraints."""
        # Constraint 1: Privacy
        if self.tag_2x2.cell == "A22" and self.tag_4x4.persistence_stage in ("PATH","CRYSTAL"):
            return False
        # Constraint 2: Process vs persistence
        if self.tag_3x3.temporal_role == "INPUT" and self.tag_4x4.persistence_stage != "EPHEMERAL":
            return False
        # Constraint 3: CL compatibility
        if self.tag_2x2.cell == "A22" and self.tag_3x3.temporal_role != "INPUT":
            return False
        return True
    
    @property
    def routing_key(self) -> str:
        """Compact routing address for this meta-position."""
        return f"{self.tag_2x2.cell}::{self.tag_3x3.cell}::{self.tag_4x4.cell}"
```


## PART 8 — CL/TIG Integration Block

### The Precise Relationship

The 2×2, 3×3, and 4×4 are **state spaces** for memory objects. CL[10×10] is an **operator grammar** that acts on transitions within the 3×3.

**Law 1:** CL[10×10] is not a replacement for these matrices. It acts on them.

**Law 2:** The 2×2 provides the TYPE of any information object. CL does not change this type — it is invariant under CL operations. You cannot CL-transform A22 (private content) into A11 (internal structure) without an explicit abstraction step.

**Law 3:** The 3×3 process table is the arena where CL acts. Each CL operation corresponds to a 3×3 cell transition:

```
CL_op[generators, lens, operator_tag] : B_{2x} → B_{3x} × write_flag

Specifically:
  fuse(G_i, G_j) -> selects which B21/B22/B23 cell is activated
  lens=TSML      -> B21→B23 (signal extraction → memory write, measurement mode)
  lens=BHML      -> B22→B33 (inference → deep memory, transformation mode)
  lens=Doing     -> B22→B31 (inference → action, execution mode)
  
  operator_tag=COMMIT  -> write_flag=True  (B23/B33 writes durable memory)
  operator_tag=DISCLAIM -> write_flag=False (discard)
  operator_tag=REGRESS -> write_flag=False + confidence -= 0.1
  operator_tag=REJECT  -> write_flag=False + mark as noise
```

**Law 4:** The 4×4 persistence state is updated by the outcome of CL operations, not by CL itself. Specifically:
- A COMMIT at B23 increments the atom's `C23` stability score (recurrence_count)
- Promotion from C21 to C31 happens when the stability score crosses the threshold
- Crystal formation (C41) happens when promotion_score(path) >= 0.85

**Diagram:**

```
WORLD                 2x2              3x3 (CL operates here)               4x4
event ──────────> classify ──────> B11 → B21 → B22 → B23 → B33 ──────> C1x → C2x → C3x → C4x
                   (A_ij)                  ↑                             (persistence update)
                                      CL[generators,
                                       lens,
                                       operator]
```

**Law 5:** The TIG force operators (G1..G9) are the generators of the CL action on the 3×3. The five force operators (G1-G5: F domain) select TRANSFORM row targets. The four structural operators (G6-G9: S domain) modify the memory depth (SIGNAL/MODEL/MEMORY column).

## PART 9 — First Implementation Target

### The Meta-Memory Event Object

For any perception event $e_t$, CK must compute:

```python
def classify_event(event: PerceptionEvent) -> MetaMemoryCoord:
    """
    Compute (P_2, P_3, P_4) for a raw perception event.
    This is the first meta-memory object.
    """
    # STEP 1: Classify 2x2 cell
    source_side = "EXTERNAL"
    semantic_side = "CONTENT" if event.has_raw_text_or_pixels else "STRUCTURE"
    privacy = "PRIVATE" if semantic_side == "CONTENT" else "SHARED_IF_ABSTRACT"
    
    # STEP 2: Assign 3x3 position (entry point for all new events)
    temporal_role = "INPUT"
    processing_layer = "SIGNAL"
    
    # STEP 3: Initial 4x4 state (all new events start EPHEMERAL)
    persistence = "EPHEMERAL"
    evaluation = "IDENTITY"
    
    # STEP 4: Compute CL point
    generators = rule_based_generator_extract(event)
    lens = assign_lens(event)
    cl_point = fuse(generators)
    
    # STEP 5: Compute routing key
    dbc27 = compute_dbc27(generators, lens)
    meta_routing_key = f"A22::B11::C11::{dbc27}"
    
    # STEP 6: Check promotability
    is_novel = novelty_gate(event, dbc27)
    promotable = len(generators) > 0 and not is_duplicate(event.content_hash)
    
    return MetaMemoryCoord(
        tag_2x2=Tag2x2(source_side, semantic_side, privacy),
        tag_3x3=Tag3x3("INPUT", "SIGNAL", "DETERMINISTIC", False, False),
        tag_4x4=Tag4x4("EPHEMERAL", "IDENTITY", confidence_seed(generators), 
                       "ELIGIBLE" if promotable else "STABLE"),
        cl_point=cl_point,
        dbc27_key=dbc27,
        meta_routing_key=meta_routing_key,
        is_novel=is_novel,
        promotable=promotable
    )
```

The output of this function is the **first meta-memory object** — a structured coordinate describing what the event is, where it is in CK's processing, and whether it should be promoted.

## Collaborator Paragraph

The 2×2/3×3/4×4 base structure defines the minimal finite grammar for CK's information-of-information layer. The 2×2 (ontological split: source × register) provides the exact privacy boundary already present in CK but now made explicit as a type constraint: raw external content (A22) is always private and never crystallizes; only abstracted structure (A21) and CK-native objects (A11, A12) can reach the crystal store. The 3×3 (process grammar: temporal role × processing depth) makes the 14-edge directed flow graph of CK's processing explicit and provides the formal gate for DeepSeek: the MODEL column (B12, B22, B32) is the only locus of probabilistic inference; all other cells are deterministic. CL[10×10] acts as the operator grammar on this process table — fuse(generators) selects the transform cell, lens selects the transition direction, operator_tag controls the memory write. The 4×4 (stabilization grammar: persistence stage × evaluation mode) formalizes the atom/path/crystal hierarchy with the stability/utility distinction that MemoryOS and RGMem independently identified as essential: a crystal can be stable but not useful (dead weight), or rare but high-utility (precious), and pruning requires both dimensions.
