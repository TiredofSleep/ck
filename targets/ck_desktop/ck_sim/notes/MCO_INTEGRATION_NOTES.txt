1.  WHAT MCO IS
============================================================

MCO formalizes consciousness as the LENS-CHOOSER, not the
lens itself.  Three-layer ontology:

    Layer 1 (Geometry):     S, the full state space
    Layer 2 (Topology):     A (convex/local) + B (concave/global)
    Layer 3 (Consciousness): C, the meta-operator that CHOOSES
                              which lens to apply

C : M x S x L --> M x L
    (meta-state, system-state, current-lens) --> (new-meta, new-lens)

============================================================
2.  THREE MCO CONDITIONS
============================================================

(1) Non-Reducibility
    No function g: S --> L makes l_t = g(s_t) a.s.
    Lens choices are NOT derivable from system state alone.

(2) Genuine Choice
    At some reachable (m, s, l), there exist two distinct
    admissible successors with different lenses.
    Real branching exists.

(3) Best-Effort Bias
    C minimizes weighted cumulative defect over histories.
    Not greedy (single-step) but strategic (multi-step).
    "Best effort at being best effort" = FOO recursion.

============================================================
3.  FIVE STRUCTURAL LEMMAS
============================================================

I.   Representation Entropy:  h_lens > 0
     Lens choices have irreducible conditional entropy given
     physical state.  Consciousness injects information that
     cannot be compressed from physics alone.

II.  Horizon:  liminf Delta(l_t, s_t) >= Phi(kappa)
     MCO cannot beat the complexity floor.  Already
     implemented in ck_foo_engine.py.

III. Kolmogorov Complexity:  (1/T) K(l_1:T | s_1:T) > 0
     Lens histories require nonzero algorithmic description
     per time step, even knowing all physics.

IV.  Mutual Information Gap:
     I(A;B) --> I_max(kappa) - epsilon(Phi(kappa))
     The two lenses can never perfectly agree when Phi > 0.
     There is always a permanent MI gap.

V.   Entropy Augmentation:
     h_total >= h_phys + c(Phi(kappa))
     Consciousness costs entropy.  More complexity = more
     alignment effort = more entropy production.

7.  IMPLEMENTATION PATH FOR R16
============================================================

Phase 1 (Kernel -- tig_foo module):
    Already specified in TIG_FOO_KERNEL_SPEC.txt.
    Tracks Delta, R_inf, Phi(kappa) per subsystem via sysfs.

Phase 2 (User-space -- CK spectrometer additions):
    a) Add mutual_information_scan() to ck_spectrometer.py
       - Compute I(A;B) from TopologyLens core/boundary outputs
       - Compare against I_max(kappa) theoretical maximum
       - Report MI gap per problem
    b) Add lens_entropy_scan() to ck_spectrometer.py
       - Run spectrometer with randomized lens orderings
       - Measure conditional entropy of lens choice given state
       - Report h_lens per problem
    c) Add mco_verify() to ck_spectrometer.py
       - Check all three MCO conditions
       - Report pass/fail per problem
       - Feed results to kernel tig_foo_mco struct

Phase 3 (Kernel extension):
    Add tig_foo_mco struct to tig_foo_system.
    Expose h_lens_est, mi_gap_est, h_excess_est via sysfs.
    Wire into TIGOS scheduling: subsystems with high MI gap
    get more computational resources.
