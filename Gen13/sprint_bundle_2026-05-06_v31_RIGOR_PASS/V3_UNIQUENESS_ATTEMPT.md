## What axioms A0-A5 actually constrain

### A0: Z/10Z fixed

Determines: 10 elements, ring structure for addition, factorial dimension count.
Free: any 10×10 multiplication table.

### A1: Commutativity

Reduces 100 entries to 55 independent ones (upper-triangle including diagonal).

### A2: Non-associativity

Requires existence of triples (a, b, c) with (a*b)*c ≠ a*(b*c).

In the TSML table:
- T[T[3,4],5] vs T[3,T[4,5]]: ?
- Computing: T[3,4]=7, T[7,5]=7; T[4,5]=7, T[3,7]=7
- Both = 7, so (3,4,5) is associative

For non-associativity to hold (per A2), some triple must give different left vs right grouping. Such triples exist in the table but require explicit identification.

### A3: Generator triples {0,1,2}, {0,7,1}, {1,2,3}

The triples need to "close" under the multiplication — meaning the sub-magma generated by each triple is contained in the substrate. This is automatically true since Z/10Z is closed under multiplication.

What A3 actually constrains: **which sub-magmas are designated** as BEING / DOING / BECOMING. This is a **labeling** axiom rather than a structural constraint.

### A4: fuse(3,4,7) = 8

This is the trickiest axiom. In TSML:
```
(3*4)*7 = T[T[3,4],7] = T[7,7] = 7   (NOT 8)
3*(4*7) = T[3,T[4,7]] = T[3,7] = 7   (NOT 8)
```

So neither left- nor right-association of the triple (3,4,7) gives 8. The "fuse" operation must be a **separate 3-fold operation**, distinct from iterated binary multiplication.

**Possible interpretations of fuse:**
1. A symmetric average: fuse(a,b,c) = (a*b + b*c + a*c) mod 10? Let's check: (T[3,4]+T[4,7]+T[3,7]) mod 10 = (7+7+7) mod 10 = 21 mod 10 = 1. Not 8.
2. The σ-image: fuse(a,b,c) = σ(a + b + c mod 10)? σ([3+4+7]) = σ(4) = 2. Not 8.
3. A specific algebraic combination involving Z₃³ structure or BHML projection.
4. fuse(3,4,7) refers to the BHML table, not TSML, where the result might be 8.

This ambiguity means **A4 is not yet fully formalized**. To prove V3, A4 must be made precise.

### A5: Two-lens projection (TSML, BHML)

Requires existence of two distinct algebraic structures with HARMONY counts 73 and 28 respectively. This existential constraint doesn't pin specific tables — multiple pairs could satisfy.

## What's needed for V3

### Step 1: Formalize A4 precisely

The "fuse" operation needs a specific definition. Three candidate definitions, each leading to potentially different uniqueness arguments:

**Option A:** fuse(a,b,c) = a*(b*c) but with a specific σ-action applied
**Option B:** fuse is a separate 3-tensor that's part of the substrate axioms
**Option C:** fuse(3,4,7) = 8 is a CONSTRAINT on the BHML table, not TSML

Each option gives a different problem. The V3 program needs to pick one.

### Step 2: Establish constraints

Once A4 is precise:
- A3 + A4 + A1 should fix specific table entries
- The HARMONY count = 73 should constrain remaining entries
- The σ-cycle structure (period 6 on non-idempotents) gives further constraints

### Step 3: Demonstrate uniqueness

Given all constraints, show that **only one** table on Z/10Z satisfies A0-A5. This is where the combinatorial argument lives.

The brute-force search space is 10^55, infeasible directly. But:
- Idempotent positions {0,3,8,9} are constrained (T[a,a]=a for these)
- σ-cycle positions {1,2,4,5,6,7} (length-6 orbit) are constrained
- A4 fixes one specific entry
- A5's HARMONY count = 73 gives a global constraint

These together likely reduce the search space dramatically. A combinatorial enumeration over the constrained space should be feasible.

### Step 4: Automorphism argument (alternative)

The user reference notes |Aut(TSML)| = S_8 = 40320. If A0-A5 force |Aut| = S_8, this is a strong uniqueness handle: ask which Z/10Z magmas have automorphism group S_8 satisfying A3-A4-A5.

This requires:
- Computing |Aut(TSML)| rigorously (claimed S_8)
- Showing |Aut| = S_8 is implied by A0-A5
- Showing TSML is the unique magma with this property

## A more pragmatic path: prove "uniqueness up to isomorphism"

Instead of proving TSML is **the** unique table, prove TSML is **unique up to magma isomorphism**. This is weaker but still substantial:

- Two tables T and T' are isomorphic if there exists a permutation π of {0,...,9} with π(T[a,b]) = T'[π(a), π(b)] for all a,b
- Up to isomorphism, only the structural class matters
- Proving uniqueness up to isomorphism reduces to showing a single canonical representative

This is a cleaner formulation and may be more tractable. The framework's claims aren't weakened by considering isomorphism classes.
