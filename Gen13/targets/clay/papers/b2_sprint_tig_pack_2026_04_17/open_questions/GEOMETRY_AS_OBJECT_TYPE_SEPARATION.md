# The Geometry Is Unfolding as Object-Type Separation, Not as One Universal Shape
## Short Note

---

Seven sprints of disciplined pre-registration have revealed something the program did not start out assuming: **the right level of description for this investigation is object-type taxonomy, not universal structure**.

When the transport program started, the implicit hope was that a single structural grammar — a unified shape — would apply across all carriers and all settings. Sprint 21's early success with cross-carrier invariants (the max-odd-unit rule, the forced closure corridor) seemed to support that hope. One grammar, many rings, common patterns.

What subsequent sprints showed, each in a different way, is that the *objects* within the program are not all of the same kind. The published Z/10 TSML is a designed artifact with proven properties under one attractor convention. The family of $C_0$ constructions under a different attractor convention is a different kind of object — an algorithmic extension, not a theorem. Noise-union seams and planted-recovery seams are both called "seams" but behave entirely differently: one grows with carrier size under sampling variance, the other stays small and structured under planting. A persistent seam at $\pi = 0.50$ and a per-seed seam at $r = 3$ are nominally the same type but carry different epistemic weight by construction.

Each sprint failure was informative precisely because the scope discipline made the object types legible in retrospect. S28 and S29 tested transport properties on a scalar summary (basin ratio) using metrics that would have worked for curves but not for scalar distributions. S30 and S30b tested extraction on objects the generator was too quiet to produce. S31-pilot-v1.0 and P3-BridgeA compared objects across paths or across generator types without appropriate bridging. In each case, the sprint's execution was faithful; the mismatch was that the chosen metric presumed a kind of object the input did not actually provide.

The deeper observation is that "structure transports" has become a precise question, not a unified hope. Some things transport within Path 2 — corridor closure and shell-partition shape are both confirmed across 23+ carriers. Some things do not transport — basin-ratio shape and smoothness-based seam topology have been closed. Whether things transport across paths is a different question entirely, with its own object-type requirements, and Bridge A's FAIL clarified that noise-union seams are not the right Path 2 input for topology-level comparisons against Path 1 designed artifacts.

This is not a failure of ambition. It is an unfolding of the geometry. What was one hypothesized shape ("a universal grammar underlies all of this") has resolved into several specific object types with distinct behaviors. The program's task has shifted: rather than hunting for the one shape that applies everywhere, the work is to understand which objects share which behaviors, under which mappings, with which epistemic status. That is taxonomy work. It is less glamorous than unified-theory work but it is what the evidence supports.

The atlas, the comparison law, and the sprint selector make this taxonomy explicit. Future sprints become easier to design cleanly and harder to fail from avoidable category errors. The program gains less "one grand result" potential and more "reliable accumulation of specific findings" potential. For a project on a multi-year horizon whose goal is to produce trustworthy transport claims rather than provocative syntheses, that trade is correct.

The next question is not "what transports?" in the abstract. It is "given these specific objects, which of them can be meaningfully compared, and what do the allowed comparisons say?" That question is answerable. It is the question the atlas sets up and the sprint selector gates. The geometry has resolved into this shape: multiple object types, explicit paths, bridging rules required whenever objects cross types, and each sprint narrow enough that its failure (or success) has clean attribution.

That is enough structure to keep building on. It is also honest structure — it does not promise more than the evidence supports, and it is organized to catch further object-type confusions before they produce spurious results.
